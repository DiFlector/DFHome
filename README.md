# DFHome

**DFHome** — локальный, приватный хаб умного дома, альтернатива Home Assistant.
Подключайте устройства по WiFi, Bluetooth, Matter, Zigbee и другим протоколам,
управляйте ими локально, стройте визуальные планы дома с функциональными
устройствами и собирайте дашборд из виджетов — от температуры и погоды в регионе
до плеера Яндекс Музыки.

DFHome **не монолит**: это небольшое ядро, в которое добавляются **интеграции**.
Всё, что специфично для протокола или вендора, живёт в интеграциях, а не в ядре.
Интеграции устанавливаются из магазина по модели HACS (Git/GitHub + курируемый
индекс).

## Ключевые идеи

- **Ядро + интеграции.** Ядро не знает про конкретные протоколы и вендоров.
- **Мультипротокольность.** WiFi / BLE / Matter / Zigbee — реализованы как
  интеграции-адаптеры. Цель: «воткнул донгл — устройства начинают поддерживаться».
- **Локальное управление.** Облако не обязательно для базовой работы.
- **План дома и виджеты.** Функциональная раскладка устройств и настраиваемый
  дашборд.
- **Магазин интеграций.** Установка из Git-репозиториев, версионирование,
  пользовательские репозитории.
- **Standalone.** Цель — запуск на Windows / Raspberry Pi / Linux, не только
  через Docker.

## Стек

- **Backend** — Python / FastAPI.
- **Frontend** — React / Vite / TypeScript (SPA).
- **Хранилище** — SQLite в каталоге данных `/data`.

## Быстрый старт (dev)

```bash
# backend (uv — https://docs.astral.sh/uv/)
cd backend
uv pip install --system -r requirements.txt
uvicorn app.main:app --reload

# frontend (pnpm)
cd frontend
pnpm install
pnpm run dev   # проксирует /api на localhost:8000
```

Запуск через Docker:

```bash
docker compose build && docker compose up -d
```

Приложение доступно на **http://localhost:1285**.

## Про `legacy/`

Папка `legacy/` — **архив прошлого подхода**: Yandex-центричный веб-хаб/прокси
для Яндекс Умного дома. Это видение устарело; теперь Яндекс — одна из интеграций.
`legacy/` сохранён как референс для переиспользования наработок и не развивается
как основной код.
